# jira
Jira is a project management and issue-tracking tool used to plan, assign, track, and manage software development work.
### A team creates a Jira ticket:
```
Issue: AUTH-101
Title: Login API returns 500
Priority: High
Assignee: Ayush Kumar Gupta
Status: In Progress
Due Date: Tomorrow
```
## In a typical software company
```
Product Manager
      ↓
Creates Story in Jira
      ↓
Sprint Planning
      ↓
Developer picks task
      ↓
Coding
      ↓
Commit code to Git
      ↓
Create Pull Request
      ↓
Code Review
      ↓
QA Testing
      ↓
Done
```
## Common Jira issue types
🐞 Bug
✨ Story (new feature)
📋 Task
🚀 Epic (a large feature made up of multiple stories)
🔧 Sub-task
## What developers do in Jira
```
View assigned tasks
Update task status
Add comments
Attach screenshots or logs
Link pull requests
Log work (if required)
Track sprint progress
```
Jira asked in interviews
```
1."Have you worked in an Agile team?"
Answer- Yes. During my academic and personal projects, I followed Agile principles by breaking the project into smaller tasks, planning features in iterations, and reviewing progress regularly. While it wasn't a large corporate Agile environment, I collaborated with teammates using Git and followed an iterative development approach similar to Scrum.
2."How did you track your tasks?"
Answer:
I tracked my work by breaking the project into small tasks, prioritizing them, and updating their status as I progressed. For team projects, I used task boards to move work through stages like To Do, In Progress, Code Review, and Done. This helped us stay organized and monitor project progress.
3."Have you used Jira?"
Answer (if you have basic experience):
Yes. I have used Jira to create and manage tasks, update issue statuses, assign work, and track project progress using Scrum boards. I'm comfortable working with stories, tasks, bugs, epics, and sprint boards.
4"What is a sprint?"
A sprint is a fixed time period, usually one to four weeks, during which an Agile team completes a set of planned tasks or user stories. At the end of the sprint, the team reviews the completed work, gathers feedback, and plans the next sprint.
5."What is a user story?"
A user story is a short description of a feature written from the end user's perspective. It describes who wants the feature, what they need, and why they need it.
```
